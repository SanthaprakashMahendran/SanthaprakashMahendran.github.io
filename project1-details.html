Real-Time Linux Server Monitoring Dashboard

My Role (DevOps):

  • Created a Bash script to collect CPU and memory usage data from Linux servers and store it in an RRDTool time-series database.
  • Scheduled data collection using cron jobs to run every 10 minutes across multiple servers.
  • After the application was developed by the team, I:
    • Designed and configured a full CI/CD pipeline using Jenkins to automate build, testing (SonarQube), and deployment processes.
    • Used Git as the version control system.
    • Containerized the application using Docker and pushed the image to AWS ECR.
    • Helm chart is used for managing Kubernetes manifests and simplifying deployment configuration.
    • Deployed the application to Kubernetes, ensuring high availability and scalability.
    • Integrated ArgoCD for GitOps-based continuous deployment to the production cluster.
    • Configured Prometheus and Grafana for real-time monitoring of application performance and health metrics.

Website Features (Developed by Dev Team):

  • A real-time, web-based dashboard to display CPU and memory usage graphs across enterprise Linux systems.
  • Search functionality to retrieve performance data by date and time, enabling faster root cause analysis during P1/P2 incidents.
  • LDAP-based authentication to restrict dashboard access to authorized users only.
