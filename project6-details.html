Precheck & Postcheck Script for Linux Server Patching

Challenge:

  • Traditionally, pre checking on Linux servers required manual validation of file system (FS) space and kernel availability. If FS usage exceeded 85%, manual cleanup was necessary. Additionally, servers had to be re-registered with Satellite to fetch the latest kernel. This process took around 5 minutes per server ,increasing manual workload significantly.

Solution:

  • Developed a script to check file system usage on Unix servers, perform cleanup if thresholds are exceeded also validates kernel versions, and re-registers servers with Satellite when needed.
  • Post-patch validation confirms kernel update success, significantly reducing human error and patch cycle time.
  • Reduced manual effort from approximately 50 hours to 5 hours per patching cycle.
